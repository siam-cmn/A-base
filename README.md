# SpecKeeper (仮)
「言った言わない」を撲滅する、仕様管理・承認プラットフォーム（API）

開発現場における「最新の仕様書がどれか分からない」「実装とドキュメントが乖離している」という課題を解決するためのバックエンドシステムです。

## 🚀 プロジェクトの目的
エンジニア・QA・PMが常に「単一の正解（Source of Truth）」を参照できる環境を提供し、コミュニケーションコストと手戻りを最小化します。

## 🛠 使用技術
- **Backend:** Laravel 12 / PHP 8.4
- **Infrastructure:** Docker (Laravel Sail 導入予定)
- **Architecture:** REST API (JSON Response only)
- **Database:** PostgreSQL

## 💎 主な機能
### 1. 組織・ユーザー管理 (Multi-tenant)
- **招待制ユーザー登録:** 組織オーナーによる招待機能。
- **一括登録:** CSVアップロードによるユーザー作成。(未実装)
- **役割ベースのアクセス制御):**
    - **Owner:** 組織管理・メンバー招待・仕様の最終承認。
    - **Admin:** 仕様の作成・編集・変更申請。
    - **Member:** 仕様の閲覧専用（QA担当者などを想定）。

### 2. 案件情報管理 (CRUD)
- 案件（プロジェクト）ごとのユーザーの紐付けによるアサイン状況管理

※以下未実装
### ３. 仕様情報管理 (CRUD)
- 案件（プロジェクト）ごとの管理。
- **画面仕様 / API仕様 / バリデーションルール**のデータ化。
- 将来的な「権限・状態マトリクス図」のデータ保持を考慮した設計。

### ４. 承認ワークフロー
- 仕様変更時の「申請 → 承認/差戻し → 反映」のステータス管理。
- 「いつ、誰が承認した正解か」を明確にする履歴保持。
